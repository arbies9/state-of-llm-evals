# Per-tool briefs

One file per tool. Each follows the same shape so you can scan them side by side:

1. **Pitch** — what the tool actually is, in one paragraph
2. **Setup cost** — what it takes to get a passing eval in CI
3. **Primitives** — the core building blocks (dataset, eval, scorer, suite, etc.)
4. **Where it shines**
5. **Where it bites**
6. **My rubric scores** — and the one-line reasoning for each axis
